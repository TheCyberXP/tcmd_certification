Lab: User Input Collection

Mission: In this lab you will set various keyboard options and conveniences to ease data entry.

Specification: To successfully complete this lab, you must develop an app that meets the following description:

  • App contains 5 input fields, each assigned different keyboard options (described below).
  • The active text field remains visible when focused and is not covered by the soft keyboard.
  • The window has a click event listener that will blur() all the fields to hide the keyboard.
1. Create a new Titanium Alloy project, giving it a name and app ID of your choice.
2. To the index view, add five text fields with ids normal, numeric, web, email, and password. Put them inside a scroll view.
3. In the index style, create style rules for your UI components. For all of the fields, set the width to 85% and height to 40dp, set the borderStyle to Ti.UI.INPUT_BORDERSTYLE_ROUNDED. Specify appropriate hintText for each of the fields. Then, set options for the fields as described:
Field Properties
normal
  • top = 50dp
  • keyboard type = default
  • return key type = default
  • autocorrect = true
  • autocapitalization = words
numeric
  • top = 150dp
  • keyboard type = numbers/punctuation
  • return key type = 'Done'
web
  • top = 250dp
  • keyboard type = URL
  • return key type = 'Go'
email
  • top = 350dp
  • keyboard type = email
  • return key type = 'Send'
password
  • top = 450dp
  • keyboard type = default
  • autocorrect = false
  • autocapitalization = none
  • password mask = true
4. To the index controller, add a click event listener on the window. When the window is clicked, blur each of the text fields to hide the soft keyboard.
5. Build your app for the simulator/emulator.
When you tap in a field, the appropriate keyboard type should be used, as should the correct return button type. Test the autocorrection and autocapitalization settings on the fields. Also, input to the password field should be obscured. Finally, the fields should remain visible when the soft keyboard is displayed.

Summary

In this lab, you implemented various keyboard and text field options that can ease data entry on the small screen.
